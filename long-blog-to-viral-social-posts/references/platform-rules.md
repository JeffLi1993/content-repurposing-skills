Last manually checked: 2026-07-20.

## LinkedIn Baseline

- Standard post text can contain up to 3,000 characters.
- Link previews use a 1.91:1 frame; LinkedIn documents 1200 x 627 pixels as a
  link-preview image size.
- Members and Pages can comment on their own posts. LinkedIn shows an `Author`
  tag on an author's own comment.
- LinkedIn's current comment help page documents commenting, replies, comment
  restrictions, and sorting. Its current visible body does not reliably
  document comment pinning even though some historical help URLs or interfaces
  have referenced it.
- LinkedIn says relevant professional content, timely insights, lessons, points
  of view, and conversation can support distribution. It advises against being
  overly promotional.

Practical defaults:

- Use one professional argument per post.
- Put the strongest claim or tension in the first 1-2 lines.
- Use short paragraphs and concrete professional examples.
- Ask a specific decision, experience, or tradeoff question.
- Use zero hashtags by default; use no more than three relevant hashtags when
  they materially aid topic context.
- Do not use a first comment as a hiding place for a bait-and-switch offer.

## X Baseline

- Standard X posts use a typical 280-character limit.
- X uses weighted character counting. Verify CJK, emoji, and URL counting when
  a draft is close to the limit.
- X Premium longer posts can extend beyond 280 characters; the official types
  page currently documents up to 25,000 characters. Do not rely on this feature
  unless the user requests it and the publishing account has access.
- Replies are separate posts in the conversation. Their timeline visibility
  can differ from the original post.
- A post beginning with a username has limited Home timeline distribution
  compared with a general post, according to X's current help page.
- X says recommendations consider relevance, credibility, safety, popularity,
  network interaction, followed accounts, and followed topics.
- X's authenticity policy prohibits manipulative behavior such as unrelated
  trend hijacking, excessive unrelated hashtags, repetitive link-only content,
  and identical or nearly identical duplicate posts.

Practical defaults:

- Use a standard post when one thesis, one proof point, and one prompt fit.
- Use a 4-7-post thread only when a framework, sequence, or story needs ordered
  expansion.
- Make the first thread post understandable and valuable by itself.
- Keep each thread item within the verified standard limit.
- Use zero hashtags by default; use no more than two when specific and helpful.
- Put the resource link in the first author reply when following this skill's
  conversion pattern.

## Pinned / First Comment Policy

Always write the CTA asset, but do not guarantee it can be pinned.

At publishing time:

1. Use the author's first comment on LinkedIn or first reply on X.
2. Pin it only when the current account and interface expose that control.
3. If pinning is unavailable, leave it as the first author comment or reply.
4. Verify that comment permissions are enabled.
5. Confirm the link destination, access requirements, and promise before
   publishing.

## Runtime Refresh Rules

- Prefer current official platform documentation.
- Record the check date in `Publishing Notes`.
- If official pages conflict, use the stricter standard-post constraint and
  disclose the uncertainty.
- Do not present third-party algorithm folklore as an official ranking factor.
- Do not claim that putting a link in the first comment guarantees greater
  reach. Use it here to create a clean post-to-resource path, not as a promised
  algorithm hack.
