# Viral Hook Template Library

Use this file as a drafting worksheet, not as a phrase bank to copy blindly.
Replace every bracketed field with a source-supported detail. Preserve any
condition attached to a number, result, or comparison.

## Hook Quality Contract

Build every Hook from:

```text
Tension + Curiosity Gap + Brevity
```

A publishable Hook must:

- Address the inferred ICP's real problem, belief, or desired result.
- Be a complete standalone sentence or compact two-line unit.
- Reveal the topic while leaving one useful question for the body to answer.
- Be repaid by the next 1-3 lines.
- Use the source's strongest specificity without exaggeration.
- Sound native in the requested language.

Prefer roughly 5-18 words in English or 12-30 CJK characters when the idea
allows it. Clarity matters more than hitting a count.

## Six High-Frequency Openings

Use this quick map when the source clearly supports one of the most repeatable
opening structures:

| Opening | Use | Template family |
|---|---|---|
| Contrarian challenge | Reverse a widely held belief | 1. Contrarian Reversal |
| Specific input + contrast result | Make a bounded outcome concrete | 17. Specific Input / Outsized Outcome |
| List / resource roundup | Promise reusable discovery value | 18. Resource Roundup |
| Before-After-Bridge | Turn a transformation into a compact story | 6. Before / After / Bridge |
| Mistake callout | Make an avoidable loss visible | 19. Mistake Callout / Loss Aversion |
| Named framework | Reduce cognitive load with a memorable system | 20. Named Framework / Mnemonic |

Do not default to these six when another family fits the source better. A
familiar opening works only when its proof and payoff are specific.

## Template Families

### 1. Contrarian Reversal

Use when the source directly challenges a common belief.

```text
[Common belief] is not what causes [desired result]. [Real cause] is.
Most [ICP] optimize [common lever]. The source shows the bottleneck is [real lever].
大家以为 [常见认知]，真正决定 [结果] 的却是 [真实因素]。
```

Proof gate: The source must explain the real cause or boundary, not merely state
an opinion.

### 2. Wrong Question

Use when changing the question improves a decision.

```text
Stop asking "[common question]." Ask "[better question]" instead.
The wrong question in [domain] is "[common question]."
做 [任务] 时，最该先改的不是答案，而是这个问题：[better question]。
```

Proof gate: The Value Content must answer the better question.

### 3. Hidden Cost

Use when an ordinary practice creates a non-obvious loss.

```text
The expensive part of [activity] is not [visible cost]. It is [hidden cost].
[Common shortcut] saves [surface benefit] and quietly costs [consequence].
[常见做法] 真正昂贵的不是 [表面成本]，而是 [隐藏代价]。
```

Proof gate: The source must identify the mechanism of the hidden cost.

### 4. Hidden Cause

Use when the visible symptom is not the root problem.

```text
[Visible problem] is usually a symptom of [root cause].
If [undesired result] keeps happening, inspect [overlooked cause] first.
[表面问题] 往往不是问题本身，而是 [根因] 的结果。
```

Proof gate: The source must connect cause and symptom.

### 5. Data-Led Surprise

Use when the source contains a strong, bounded number.

```text
[Number] sounds like success—until you compare it with [second fact].
Only [number] of [sample/context] achieved [result]. The reason matters more than the number.
[条件] 下，[数据] 的真正含义不是 [常见解读]，而是 [新解读]。
```

Proof gate: Preserve the exact number, unit, sample, source, date, and relevant
condition. Never manufacture precision.

### 6. Before / After / Bridge

Use when a case or experiment contains a painful before state, a credible
turning point, and a materially different after state.

```text
We changed one part of [process]: [before] -> [after].
[Before state] looked efficient. [After insight] changed the result.
[Before state]. Then [turning point]. Now [after state]. The bridge was [mechanism].
[痛苦现状]。直到 [转折点]。现在 [结果]。真正的桥梁是 [方法/机制]。
```

Proof gate: The source must contain all three beats. Name the Bridge instead of
jumping directly from pain to success. Do not imply causation beyond the source.

### 7. Mistake Confession

Use when the source includes a real error and lesson.

```text
I spent [bounded cost] solving the wrong [problem].
Our [initiative] failed for a reason I did not expect: [reason].
我们在 [任务] 上犯过一个很贵的错：[具体错误]。
```

Proof gate: Use first person only when the source or publishing account can
truthfully own the experience.

### 8. Specific ICP Callout

Use when a narrow audience faces a recognizable situation.

```text
If you are a [specific ICP] still [behavior], this is the risk to watch.
[ICP] usually discover [problem] one step too late.
如果你是正在 [具体场景] 的 [ICP]，先别急着 [常见动作]。
```

Proof gate: The source must support the audience, risk, and scenario.

### 9. Tradeoff

Use when the best decision depends on what must be sacrificed.

```text
You can optimize [A] or [B] first—but not both under [constraint].
[Choice A] wins on [benefit]. [Choice B] wins when [condition].
[A] 和 [B] 不是谁更好，而是你愿意先牺牲什么。
```

Proof gate: The body must name the decision rule or condition.

### 10. Framework Promise

Use when the source contains a repeatable model.

```text
Use this [number]-part test before you [high-stakes task].
A useful [process] needs only [number] decisions.
做 [任务] 前，先过这 [number] 个判断。
```

Proof gate: The exact framework must appear in the body. Do not add a number
that the content cannot fulfill.

### 11. Checklist / Save Utility

Use when the post can become a reusable reference.

```text
Save this checklist for the next time you [specific task].
Before you [task], check these [number] failure points.
下次做 [任务] 前，先保存这份 [具体清单]。
```

Proof gate: The body must provide a complete, compact checklist worth saving.

### 12. Experiment / Result

Use when the source documents a test and a bounded outcome.

```text
We tested [A] against [B] under [condition]. [Result] changed our view.
One [duration/sample] experiment exposed the real bottleneck in [process].
我们用 [条件] 测了 [A] 和 [B]，结果推翻了一个原来的判断。
```

Proof gate: State the setup, result, and limitation. Avoid universal claims from
one test.

### 13. Counterintuitive Sequence

Use when doing the usual steps in a different order matters.

```text
Do [unexpected first step] before you [usual first step].
The fastest route to [result] starts by removing [thing], not adding [thing].
想更快得到 [结果]，第一步不是增加 [动作]，而是删掉 [阻力]。
```

Proof gate: The source must explain why the sequence works.

### 14. Operator Lesson

Use when the source contains a practical judgment earned through execution.

```text
The operator lesson from [experience]: [compact judgment].
[Practice] looks good in a deck and breaks at [real-world constraint].
一个只有真正做过 [任务] 才会遇到的问题：[具体问题]。
```

Proof gate: Tie the lesson to a concrete example, constraint, or observed
failure.

### 15. Story Scene

Use when a specific moment creates immediate tension.

```text
At [specific moment], [unexpected event] exposed [problem].
The plan looked finished until [turning point].
[具体时刻]，一个 [意外信号] 让我们重新检查了整个 [流程]。
```

Proof gate: Use only source-grounded scenes. Do not fictionalize dialogue or
details.

### 16. Timely Change

Use only when a verified recent development creates a direct reason to care.

```text
[Recent change] makes one old [domain] assumption obsolete.
After [verified update], [ICP] need to rethink [specific decision].
[近期变化] 之后，[ICP] 最该重做的不是 [表面动作]，而是 [关键决策]。
```

Proof gate: Verify the change, event date, source date, and bridge to the
article. Cite it in delivery notes.

### 17. Specific Input / Outsized Outcome

Use when a bounded amount of time, money, people, or effort produced a
surprising but source-supported result.

```text
I used [specific input] to produce [specific result] under [condition].
[Small input] -> [large result]. The unexpected lever was [factor].
用 [具体时间/成本/资源] 做到 [具体结果]；真正起作用的是 [关键因素]。
```

Proof gate: Preserve the exact input, result, conditions, date, and account
ownership. Do not imply that every reader can reproduce the result.

### 18. Resource Roundup

Use when the source contains a genuinely useful collection, tool list, reading
path, prompt set, dataset, or directory.

```text
[Number] [resources] for [specific ICP task]—including [most surprising item].
[Number] resources most [ICP] miss when [scenario].
[Number] 个适合 [ICP] 做 [任务] 的资源，其中 [具体资源] 最容易被忽略。
```

Proof gate: Deliver every promised item or an honest representative subset with
a clear path to the full list. Use `I use this` only when the author or
publishing account can truthfully own that experience.

### 19. Mistake Callout / Loss Aversion

Use when a common behavior causes a concrete, avoidable loss.

```text
If you still [specific behavior], you are losing [bounded consequence] before [stage].
[ICP], stop [mistake]. It quietly creates [specific loss].
你不是缺少 [能力/资源]，而是一直在 [具体错误] 上损失 [结果]。
```

Proof gate: Name the behavior, mechanism, and consequence. Use a number only
when the source supports it. Challenge the mistake, not the reader's worth or
identity.

### 20. Named Framework / Mnemonic

Use when a short name makes a real multi-step method easier to remember and
apply.

```text
The [NAME] framework: [step A], [step B], [step C].
Use [A-B-C] to [specific job]: [A meaning] -> [B meaning] -> [C meaning].
[NAME] 方法：[步骤 A]、[步骤 B]、[步骤 C]，用来解决 [具体任务]。
```

Naming rules:

- Use 2-5 real components from the source.
- Make every letter or word map cleanly to one component.
- Prefer a short, pronounceable name over a forced acronym.
- Do not imply the framework is proprietary, established, or validated unless
  the source supports that claim.

Proof gate: Explain every component in the Value Content. A clever name cannot
substitute for a useful method.

## Emotional Levers And Interaction Triggers

Choose one primary emotional lever and one matching interaction trigger for
each Hook. Emotion should sharpen the source insight, not manufacture drama.

| Hook situation | Primary emotional lever | Matching interaction trigger |
|---|---|---|
| Belief reversal | Surprise / constructive disagreement | Ask which belief the reader would keep or reject |
| Specific result | Awe / ambition / skepticism | Ask what lever the reader thinks mattered most |
| Resource roundup | Relief / discovery | Invite a save or ask which resource is missing |
| Before-After-Bridge | Hope / identification | Ask for a comparable turning point or obstacle |
| Mistake callout | Loss aversion / frustration | Ask which mistake is most costly in practice |
| Named framework | Relief / mastery | Ask which step the reader would apply first |
| Tradeoff | Tension / professional identity | Require a choice under a stated constraint |
| Story scene | Curiosity / empathy | Ask whether the reader has seen the same pattern |
| Data surprise | Surprise / trust | Ask for the reader's interpretation, not blind agreement |
| Checklist | Control / preparedness | Invite saving for a named future task |

Use humor only when the source, brand voice, and subject make it natural. Never
mock the ICP, a vulnerable group, or a real failure.

Do not use rage as a default growth tactic. When the source contains legitimate
frustration, translate it into a specific disputed practice, tradeoff, or
professional stance. Invite a reasoned position instead of outrage.

Keep continuity between the opening and ending:

```text
Hook tension -> Value payoff -> Matching interaction trigger
```

Do not open with loss aversion and end with an unrelated generic question. Do
not open with a save-worthy list and omit the promised list.

## Anti-Templates

Do not use:

- `Stop scrolling.`
- `You won't believe what happened next.`
- `This changes everything.`
- `Nobody is talking about this.`
- `The secret they don't want you to know.`
- `I asked AI to [task], and the result blew my mind.`
- Unverified `90% of...`, `10x...`, `overnight...`, or `guaranteed...` claims.
- A vague question such as `Are you ready for the future?`
- A numbered promise when the body does not deliver every promised item.

Do not confuse obscurity with curiosity. The reader should know the topic,
stakes, and intended audience from the Hook.
